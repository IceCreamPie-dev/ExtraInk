시스템 개요

ExtraInk v2.1은 SOLID 원칙을 적용한 완전히 재설계된 캐릭터 및 대화 시스템입니다.

v2.1 주요 개선사항

  • 고급 설정 시스템: 플랫폼별 최적화된 설정 저장소 (Registry, JSON, 암호화 파일, PlayerPrefs)
  • DI 시스템 도입: Constructor Injection 기반 의존성 주입 시스템 추가 (v2.2 준비)
  • 단일 책임: 각 매니저가 명확한 하나의 책임만 담당
  • 완전한 하위 호환성: 기존 한국어 API 메서드 유지
  • 다중 API 지원: 한국어와 영어 메서드 동시 제공
  • 안정성 개선: 컴파일 오류 수정 및 deprecated API 업데이트

주요 컴포넌트

통합 매니저 시스템

  • AudioManager: 통합 오디오 관리 (BGM, SFX, Voice, Volume 제어)
  • CharacterManager: 캐릭터 생성, 감정 변경, 애니메이션 관리
  • DialogueManager: 대화 표시, 텍스트 애니메이션, 선택지 처리
  • SettingsManager: 통합 설정 관리 (Audio, Gameplay, Language)
  • UIManager: UI 상태 관리, 애니메이션, 데이터 바인딩
  • LocalizationManager: 다국어 지원, 언어별 리소스 관리

캐릭터 시스템

  • BaseCharacterComponent: 모든 캐릭터 컴포넌트의 기본 클래스
  • SimpleSpriteCharacterComponent: 스프라이트 기반 캐릭터 구현체
  • SimpleAnimatorCharacterComponent: Animator 기반 캐릭터 구현체
  • AnimatorCharacterComponent: 고급 Animator 기반 캐릭터 구현체
  • CharacterDisplay: 기존 호환성을 위한 캐릭터 표시 클래스
  • SpriteCharacterRenderer: 전용 스프라이트 렌더러
  • ICPTweenCharacterAnimator: ICPTween 기반 애니메이터
  • TextAnimationSystem: 고급 텍스트 애니메이션 시스템
  • InkCharacterBridge: Ink External Function 연결 브릿지

플러그인 시스템

  • IExtraInkPlugin: 플러그인 인터페이스
  • BaseExtraInkPlugin: 플러그인 기본 구현체
  • PluginManager: 플러그인 생명주기 관리
  • PluginContext: 플러그인 실행 컨텍스트
  • PluginManagerWindow: 플러그인 관리 에디터 도구
  • PluginCreatorWindow: 플러그인 생성 마법사

이벤트 시스템

  • BaseEventChannel: 기본 이벤트 채널
  • EventChannelManager: 이벤트 채널 관리자
  • EventChannelListener: 이벤트 리스너 컴포넌트
  • EventChannelBridge: 기존 시스템과의 브릿지

핵심 데이터 시스템

  • CharacterData: 캐릭터 정보를 담는 ScriptableObject
  • CharacterDatabase: 모든 캐릭터를 관리하는 데이터베이스
  • AudioClipData & AudioDatabase: 오디오 관리 시스템
  • StoryData & StoryDatabase: 스토리 관리 시스템
  • DialogueStyleAsset: 대화 스타일 커스터마이징
  • StylePresets: 미리 정의된 스타일 프리셋들
  • ISettingsStorage: 플랫폼별 최적화된 설정 저장소 시스템

  • 플러그인 아키텍처: 게임 장르별 모듈형 시스템 지원

  • Ink 스토리와 원활한 연동: InkCharacterBridge를 통한 완벽한 연결
  • 확장 가능한 컴포넌트 구조: 개발자가 자유롭게 커스터마이징 가능
  • 완벽한 다국어 지원: 언어별 스토리, 오디오, 캐릭터 데이터 관리
  • 스타일 시스템: DialogueStyleAsset을 통한 UI 테마 관리
  • 고급 오디오 시스템: 카테고리별 오디오 관리와 크로스페이드
  • 플랫폼 최적화 설정: Windows Registry, JSON File, 암호화 파일, PlayerPrefs 자동 선택

  • 텍스트 애니메이션: 타이핑, 페이드, 흔들림 등 다양한 효과

  • SOLID 아키텍처: 유지보수성과 확장성을 고려한 설계
  • 이벤트 기반 아키텍처: UnityEvent 시스템으로 컴포넌트 간 안전한 통신
  • 통합 에디터 도구: ExtraInkWindow로 모든 기능 중앙 관리

DI 사용 예제

전체 사용 예제는 Assets/DeulPie/ExtraInk/Core/DI/DIUsageExample.cs에서 확인할 수 있습니다.

// 완전한 DI 기반 게임 매니저 예제
public class ModernGameManager : MonoBehaviour
{
private readonly IAudioManager _audioManager;
private readonly ICharacterManager _characterManager;
private readonly IDialogueManager _dialogueManager;
public ModernGameManager(
IAudioManager audioManager,
ICharacterManager characterManager,
IDialogueManager dialogueManager)
{
_audioManager = audioManager;
_characterManager = characterManager;
_dialogueManager = dialogueManager;
}
public void StartGame()
{
_audioManager.PlayBGM("main_theme");
_characterManager.ShowCharacter("hero", "normal", "center");
_dialogueManager.ShowDialogue("게임이 시작됩니다!", "시스템");
}
}

고급 설정 시스템

ExtraInk v2.1에서는 플랫폼별로 최적화된 고급 설정 시스템을 제공합니다. 이 시스템은 ISettingsStorage 인터페이스를 통해 다양한 저장 방식을 추상화하여 제공합니다.

설정 시스템 핵심 특징

  • 플랫폼 최적화: 각 플랫폼에 가장 적합한 저장 방식 자동 선택
  • 타입 안전성: 제네릭을 통한 강타입 설정 관리
  • 암호화 지원: 민감한 설정을 위한 암호화 저장소
  • 빌드 자동화: 빌드 시 플랫폼별 자동 구성
  • 마이그레이션: 저장소 간 설정 이전 지원

저장소 타입별 특징

1. Registry Storage (Windows 전용)

// Windows 레지스트리를 사용한 설정 저장
var storage = new RegistrySettingsStorage("MyGame");
storage.SaveSetting("volume", 0.8f);
float volume = storage.LoadSetting("volume", 1.0f);

장점: Windows 표준 방식, 시스템 통합성, 빠른 액세스 단점: Windows 전용, 관리자 권한 필요할 수 있음

2. JSON File Storage (크로스 플랫폼)

// JSON 파일을 사용한 설정 저장
var storage = new JsonFileSettingsStorage("settings.json");
storage.SaveSetting("language", "ko");
string language = storage.LoadSetting("language", "en");

장점: 모든 플랫폼 지원, 가독성, 디버깅 용이 단점: 보안성 낮음, 파일 크기 클 수 있음

3. Encrypted File Storage (보안 중요)

// 암호화된 파일을 사용한 보안 설정 저장
var storage = new EncryptedFileSettingsStorage("secure_settings.dat", "encryption_key");
storage.SaveSetting("user_token", "secret_token_value");
string token = storage.LoadSetting("user_token", "");

장점: 높은 보안성, 모든 플랫폼 지원, 민감한 정보 보호 단점: 성능 오버헤드, 키 관리 필요

4. PlayerPrefs Storage (Unity 기본)

// Unity PlayerPrefs를 사용한 설정 저장
var storage = new PlayerPrefsSettingsStorage("MyGame_");
storage.SaveSetting("music_enabled", true);
bool musicEnabled = storage.LoadSetting("music_enabled", true);

장점: Unity 기본 제공, WebGL 완전 지원, 간단한 사용법 단점: 제한적인 데이터 타입, 플랫폼별 제약사항

플러그인 시스템

ExtraInk의 플러그인 시스템을 사용하여 게임 장르별로 특화된 기능을 모듈형으로 개발할 수 있습니다.

플러그인 시스템 개요

ExtraInk 플러그인 시스템은 다음과 같은 특징을 제공합니다:

  • 장르별 특화: RPG, 연애 시뮬레이션, 미스터리 등 게임 장르별 전용 플러그인
  • Hot Reload: 에디터에서 플러그인 수정 시 실시간 재로드
  • 의존성 관리: 플러그인 간 의존성 자동 해결
  • 다국어 지원: 플러그인도 다국어 시스템과 연동
  • 성능 최적화: 필요한 플러그인만 선택적 로드

일반적인 문제 해결

매니저 초기화 순서 문제

증상: NullReferenceException이 발생하며 매니저가 제대로 작동하지 않음

해결책:

  1. ManagerBootstrapper 사용하여 매니저 초기화 순서 제어
  2. 각 매니저의 IsInitialized 프로퍼티 확인
  3. DontDestroyOnLoad 설정 확인
// 올바른 매니저 참조 방법
private void InitializeManagers()
{
// Instance가 null인지 확인
if (AudioManager.Instance == null)
{
Debug.LogError("AudioManager가 초기화되지 않았습니다!");
return;
}
audioManager = AudioManager.Instance;
// ... 다른 매니저들
}

Ink 스토리 파일 로드 실패

증상: Ink 스토리가 제대로 로드되지 않음

해결책:

  1. .ink 파일이 Resources 폴더에 있는지 확인
  2. Ink Compiler 설정 확인
  3. JSON 출력 파일이 생성되었는지 확인
// 안전한 스토리 로드 방법

}
