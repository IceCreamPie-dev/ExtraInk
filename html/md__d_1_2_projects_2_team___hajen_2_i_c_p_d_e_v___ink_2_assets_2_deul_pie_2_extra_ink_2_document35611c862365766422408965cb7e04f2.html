ExtraInk 캐릭터 시스템 사용 가이드

개요

캐릭터 시스템은 개발자가 ICharacterController 인터페이스를 구현하여 커스텀 캐릭터 기능을 자유롭게 추가할 수 있도록 설계되었습니다.

캐릭터 데이터 설정

1. CharacterData 설정

// CharacterData ScriptableObject에서 설정
characterPrefab: 커스텀 캐릭터 프리팹 (ICharacterController 구현체)
rendererPrefab: 렌더러 프리팹 (ICharacterRenderer 구현체)
animatorPrefab: 애니메이터 프리팹 (ICharacterAnimator 구현체)
ICharacterAnimator
캐릭터 애니메이션을 담당하는 인터페이스 다양한 애니메이션 시스템(ICPTween, Unity Animator, DOTween 등)을 지원합니다

ICharacterController

캐릭터 렌더링을 담당하는 인터페이스 다양한 렌더링 방식(스프라이트, 애니메이터, 3D 모델 등)을 지원합니다
Definition ICharacterRenderer.cs:8

2. 우선 순위

  1. characterPrefab이 있고 ICharacterController를 구현하는 경우 → 새로운 시스템 사용
  2. characterPrefab이 있지만 ICharacterController가 없는 경우 → 기존 시스템 사용
  3. rendererPrefab이나 animatorPrefab이 있는 경우 → 개별 컴포넌트 사용
  4. 모두 없는 경우 → 기본 프리팹 사용

2. 프리팹 생성

  1. 새로운 게임 오브젝트 생성
  2. 나의_커스텀_캐릭터_컨트롤러 컴포넌트 추가
  3. 필요한 컴포넌트들 추가 (ParticleSystem, AudioSource, SpriteRenderer 등)
  4. 프리팹으로 저장
  5. CharacterData의 characterPrefab 필드에 할당

Ink에서 사용하기

기본 사용법

//영문
# character_show:myCharacter:happy:center:fade
안녕하세요! 저는 커스텀 캐릭터입니다.
# character_emotion:myCharacter:sad
슬픈 표정으로 변경됩니다.
# character_action:myCharacter:lightning
번개 효과를 사용합니다!
# character_action:myCharacter:teleport:5:0:0
텔레포트합니다!
# character_action:myCharacter:summon
소환 마법을 사용합니다!
# character_hide:myCharacter:disappear
사라집니다!

# character_action:wizard:teleport:10:5:0
# character_action:wizard:summon:goblin:3
# character_action:princess:dance:slow
# character_action:knight:attack:sword:enemy1

장점

  1. 확장성: 개발자가 자유롭게 커스텀 기능 추가 가능
  2. 호환성: 기존 시스템과 완전 호환
  3. 유연성: Ink 태그를 통해 다양한 액션 실행 가능
  4. 재사용성: 프리팹으로 저장하여 여러 캐릭터에 재사용 가능
  5. 독립성: 각 캐릭터가 독립적인 로직을 가질 수 있음
