ScriptableObject 기반 이벤트 채널 시스템

ExtraInk의 강화된 이벤트 시스템으로, ScriptableObject를 기반으로 하는 타입 안전하고 Inspector 친화적인 이벤트 채널을 제공합니다.

🎯 주요 특징

✅ 강력한 기능들

  • 타입 안전성: 컴파일 타임에 타입 체크로 런타임 오류 방지
  • Unity Inspector 통합: Inspector에서 직접 이벤트 연결 및 설정
  • ScriptableObject 기반: 프로젝트 에셋으로 관리되는 재사용 가능한 이벤트 채널
  • 에디터 도구: 이벤트 채널 생성, 관리, 테스트를 위한 전용 에디터 윈도우
  • 하위 호환성: 기존 GameEvents 시스템과 완벽 연동
  • 성능 최적화: 효율적인 이벤트 구독/발행 시스템
  • 실시간 디버깅: 구독자 수, 이벤트 발생 로그 등 실시간 모니터링

1. 이벤트 채널 생성

// 1. CreateAssetMenu를 통해 생성

// 2. 코드로 동적 생성
var gameStartChannel = ScriptableObject.CreateInstance<VoidEventChannel>();
var playerNameChannel = ScriptableObject.CreateInstance<StringEventChannel>();

// GameObject에 StringEventChannelListener 컴포넌트 추가

코드로 구독

public class UIController : MonoBehaviour
{
[SerializeField] private StringEventChannel playerNameChannel;
[SerializeField] private Text playerNameText;
void OnEnable()
{
playerNameChannel.Subscribe(OnPlayerNameChanged);
}
void OnDisable()
{
playerNameChannel.Unsubscribe(OnPlayerNameChanged);
}
private void OnPlayerNameChanged(string newName)
{
playerNameText.text = newName;
}
}

📚 이벤트 채널 타입들

1. VoidEventChannel

파라미터가 없는 간단한 이벤트입니다.

// 사용 예시
[SerializeField] private VoidEventChannel gameStartChannel;
[SerializeField] private VoidEventChannel gamePauseChannel;
// 이벤트 발생
gameStartChannel.RaiseEvent();
// 구독
gameStartChannel.Subscribe(() => {
Debug.Log("게임이 시작되었습니다!");
});

적용 사례:

  • 게임 시작/종료
  • 일시정지/재개
  • 레벨 완료
  • 메뉴 전환

2. StringEventChannel

문자열 파라미터를 전달하는 이벤트입니다.

// 사용 예시
[SerializeField] private StringEventChannel messageChannel;
[SerializeField] private StringEventChannel sceneChangeChannel;
// 이벤트 발생
messageChannel.RaiseEvent("안녕하세요!");
sceneChangeChannel.RaiseEvent("MainMenu");
// 구독
messageChannel.Subscribe(message => {
Debug.Log($"메시지 수신: {message}");
});

적용 사례:

  • 메시지 표시
  • 씬 변경
  • 사운드 재생 (ID 전달)
  • 설정 변경

3. IntEventChannel

정수 파라미터를 전달하는 이벤트입니다.

// 사용 예시
[SerializeField] private IntEventChannel scoreChannel;
[SerializeField] private IntEventChannel levelChannel;
// 값 범위 설정 가능
scoreChannel.minValue = 0;
scoreChannel.maxValue = 999999;
// 이벤트 발생
scoreChannel.RaiseEvent(1000);
levelChannel.RaiseEvent(5);
// 구독
scoreChannel.Subscribe(score => {
Debug.Log($"점수 업데이트: {score}");
});

적용 사례:

  • 점수/경험치 변경
  • 레벨 변경
  • 아이템 수량 변경
  • 선택지 선택

4. FloatEventChannel

실수 파라미터를 전달하는 이벤트입니다.

// 사용 예시
[SerializeField] private FloatEventChannel healthChannel;
[SerializeField] private FloatEventChannel volumeChannel;
// 이벤트 발생
healthChannel.RaiseEvent(75.5f);
volumeChannel.RaiseEvent(0.8f);
// 정규화된 값으로 발생 (0-1 범위를 실제 범위로 변환)
volumeChannel.RaiseNormalizedEvent(0.5f); // 50% 볼륨
// 구독
healthChannel.Subscribe(health => {
UpdateHealthBar(health);
});

적용 사례:

  • 체력/마나 변경
  • 볼륨 조절
  • 진행도 표시
  • 시간 관련 값

5. Vector3EventChannel

Vector3 파라미터를 전달하는 이벤트입니다.

// 사용 예시
[SerializeField] private Vector3EventChannel positionChannel;
[SerializeField] private Vector3EventChannel rotationChannel;
// 이벤트 발생
positionChannel.RaiseEvent(new Vector3(10, 0, 5));
positionChannel.RaiseEvent(10, 0, 5); // 개별 값으로 발생
positionChannel.RaiseEvent2D(10, 5); // 2D 위치 (z=0)
// 구독
positionChannel.Subscribe(position => {
transform.position = position;
});

적용 사례:

  • 오브젝트 위치 변경
  • 카메라 회전
  • 스케일 변경
  • 물리 힘 적용

6. GameObjectEventChannel

GameObject 참조를 전달하는 이벤트입니다.

// 사용 예시
[SerializeField] private GameObjectEventChannel spawnChannel;
[SerializeField] private GameObjectEventChannel destroyChannel;
// 필터링 설정
spawnChannel.allowedTag = "Enemy";
spawnChannel.allowedLayer = 8;
// 이벤트 발생
spawnChannel.RaiseEvent(enemyPrefab);
destroyChannel.RaiseEvent(targetObject);
// 특정 컴포넌트를 가진 오브젝트만 허용
spawnChannel.RaiseEventWithComponent<Rigidbody>(physicObject);
// 구독
spawnChannel.Subscribe(spawnedObject => {
Debug.Log($"오브젝트 생성됨: {spawnedObject.name}");
});

적용 사례:

  • 오브젝트 생성/제거 알림
  • 타겟 변경
  • 충돌 감지 결과
  • UI 패널 활성화

7. CharacterEventChannel (특화 채널)

캐릭터 관련 복합 이벤트를 처리하는 특화된 채널입니다.

// 사용 예시
[SerializeField] private CharacterEventChannel characterChannel;
// 유효한 값 설정
characterChannel.validCharacters = new[] { "hero", "villain", "npc" };
characterChannel.validEmotions = new[] { "happy", "sad", "angry" };
characterChannel.validPositions = new[] { "left", "center", "right" };
// 이벤트 발생
characterChannel.RaiseEvent("hero", "happy", "center", "fade");
// 편의 메서드들
characterChannel.ShowCharacter("hero", "happy", "center");
characterChannel.HideCharacter("hero", "fade");
characterChannel.ChangeEmotion("hero", "sad");
characterChannel.MoveCharacter("hero", "left");
// 구독
characterChannel.Subscribe((characterId, emotion, position, effect) => {
CharacterManager.Instance.ShowCharacter(characterId, emotion, position, effect);
});

  • 캐릭터 등장/퇴장
  • 감정 변경
  • 위치 이동
  • 애니메이션 트리거

8. DialogueEventChannel (특화 채널)

대화 관련 이벤트를 처리하는 특화된 채널입니다.

// 사용 예시
[SerializeField] private DialogueEventChannel dialogueChannel;
// 텍스트 검증 설정
dialogueChannel.maxTextLength = 200;
dialogueChannel.allowHtmlTags = true;
dialogueChannel.validSpeakers = new[] { "hero", "villain", "narrator" };
// 이벤트 발생
dialogueChannel.RaiseEvent("안녕하세요!", "hero");
// 편의 메서드들
dialogueChannel.RaiseNarrationEvent("이야기가 시작되었다.");
dialogueChannel.RaiseSystemEvent("게임이 저장되었습니다.");
// 구독
dialogueChannel.Subscribe((text, speaker) => {
DialogueUI.Instance.ShowDialogue(text, speaker);
});

적용 사례:

  • 대화 표시
  • 내레이션
  • 시스템 메시지
  • 자막 표시

🎛️ EventChannelManager

모든 이벤트 채널을 중앙에서 관리하는 싱글톤 매니저입니다.

// 매니저 접근
EventChannelManager manager = EventChannelManager.Instance;
// 채널 등록
manager.RegisterChannel(myChannel);
// 채널 검색
var gameStartChannel = manager.GetChannel<VoidEventChannel>("GameStart");
var allStringChannels = manager.GetChannels<StringEventChannel>();
// 채널 존재 확인
bool exists = manager.HasChannel("PlayerHealth");
// 통계 정보
string stats = manager.GetStatistics();
Debug.Log(stats);

자동 채널 로드

// Resources 폴더에서 자동 로드
EventChannelManager.autoLoadFromResources = true;
EventChannelManager.resourcePath = "EventChannels";
// 수동 등록
EventChannelManager.manuallyRegisteredChannels = new[] {
myChannel1,
myChannel2

🔗 기존 시스템과의 통합

EventChannelBridge

기존 GameEvents 시스템과 새로운 EventChannel 시스템을 연결하는 브리지입니다.

// 양방향 이벤트 연결
public class EventChannelBridge : MonoBehaviour
{
[SerializeField] private CharacterEventChannel characterChannel;
[SerializeField] private DialogueEventChannel dialogueChannel;
void Start()
{
// EventChannel -> GameEvents
characterChannel.Subscribe((id, emotion, pos, effect) => {
GameEvents.OnCharacterShow?.Invoke(id, emotion, pos, effect);
});
// GameEvents -> EventChannel
GameEvents.OnDialogueRequested.AddListener((text, speaker) => {
dialogueChannel.RaiseEvent(text, speaker);
});
}
}

마이그레이션 가이드

// 기존 방식
GameEvents.OnCharacterShow.AddListener(OnCharacterShow);
GameEvents.OnCharacterShow?.Invoke("hero", "happy", "center", "fade");
// 새로운 방식
characterEventChannel.Subscribe(OnCharacterShow);
characterEventChannel.ShowCharacter("hero", "happy", "center");
// 하위 호환성 (EventChannelBridge 사용 시 양쪽 모두 동작)

🎯 실제 사용 예제

게임 시작 시퀀스

public class GameStartSequence : MonoBehaviour
{
[Header("이벤트 채널들")]
[SerializeField] private VoidEventChannel gameStartChannel;
[SerializeField] private StringEventChannel sceneTransitionChannel;
[SerializeField] private CharacterEventChannel characterChannel;
[SerializeField] private DialogueEventChannel dialogueChannel;
void Start()
{
StartCoroutine(GameStartCoroutine());
}
private IEnumerator GameStartCoroutine()
{
// 게임 시작 이벤트
gameStartChannel.RaiseEvent();
yield return new WaitForSeconds(1f);
// 메인 씬으로 전환
sceneTransitionChannel.RaiseEvent("MainScene");
yield return new WaitForSeconds(2f);
// 주인공 등장
characterChannel.ShowCharacter("hero", "determined", "center");
yield return new WaitForSeconds(1f);
// 시작 대사
dialogueChannel.RaiseEvent("새로운 모험이 시작된다!", "hero");
}
}

플레이어 상태 관리

public class PlayerStatsManager : MonoBehaviour
{
[Header("상태 이벤트 채널들")]
[SerializeField] private FloatEventChannel healthChannel;
[SerializeField] private FloatEventChannel manaChannel;
[SerializeField] private IntEventChannel levelChannel;
[SerializeField] private StringEventChannel statusChannel;
[Header("현재 상태")]
[SerializeField] private float currentHealth = 100f;
[SerializeField] private float currentMana = 50f;
[SerializeField] private int currentLevel = 1;
void Start()
{
// 초기 상태 이벤트 발생
healthChannel.RaiseEvent(currentHealth);
manaChannel.RaiseEvent(currentMana);
levelChannel.RaiseEvent(currentLevel);
}
public void TakeDamage(float damage)
{
currentHealth = Mathf.Max(0, currentHealth - damage);
healthChannel.RaiseEvent(currentHealth);
if (currentHealth <= 0)
{
statusChannel.RaiseEvent("dead");
}
else if (currentHealth <= 20)
{
statusChannel.RaiseEvent("critical");
}
}
public void LevelUp()
{
currentLevel++;
levelChannel.RaiseEvent(currentLevel);
statusChannel.RaiseEvent("levelup");
}
}

[CreateAssetMenu(fileName = "New Chain Event Channel",
menuName = "ExtraInk/Event Channels/Custom/Chain Event Channel")]
public class ChainEventChannel : VoidEventChannel
{
[Header("체인 설정")]
[SerializeField] private VoidEventChannel[] chainedEvents;
[SerializeField] private float[] delays;
public override void RaiseEvent()
{
base.RaiseEvent();
if (chainedEvents != null && chainedEvents.Length > 0)
{
StartCoroutine(ExecuteChain());
}
}
private IEnumerator ExecuteChain()
{
for (int i = 0; i < chainedEvents.Length; i++)
{
if (chainedEvents[i] != null)
{
float delay = i < delays.Length ? delays[i] : 0f;
if (delay > 0)
{
yield return new WaitForSeconds(delay);
}
chainedEvents[i].RaiseEvent();
DebugLog($"체인 이벤트 {i + 1}/{chainedEvents.Length} 실행됨");
}
}
}

자주 묻는 질문 (FAQ)

Q: 기존 GameEvents와 어떤 차이가 있나요? A: ScriptableObject 기반으로 Inspector에서 직접 연결 가능하고, 타입 안전성과 재사용성이 뛰어납니다.

Q: 성능에 문제는 없나요? A: 기존 시스템과 동등하거나 더 나은 성능을 제공하며, 추가적인 기능들이 있습니다.

Q: 기존 코드를 모두 바꿔야 하나요? A: EventChannelBridge를 사용하면 기존 코드 수정 없이 점진적으로 마이그레이션 가능합니다.

Q: 런타임에 채널을 동적으로 생성할 수 있나요? A: 네, ScriptableObject.CreateInstance를 사용해서 런타임에 생성 가능합니다.

Q: 여러 씬에서 채널을 공유할 수 있나요? A: Resources 폴더의 채널은 모든 씬에서 접근 가능하며, EventChannelManager가 자동으로 관리합니다.
